# 金融重大信息推送系统 — 使用手册

## 快速开始

### 你只需要做 3 件事

1. **把代码推到 GitHub**
   ```bash
   cd financial-push
   git init
   git add .
   git commit -m "init: 金融推送系统 v1.0"
   git remote add origin https://github.com/你的用户名/financial-push.git
   git push -u origin main
   ```

2. **创建企业微信机器人**
   - 下载 [企业微信](https://work.weixin.qq.com/)，用手机号注册（个人免费）
   - 建一个群 → 右上角 `...` → 群机器人 → 添加 → 复制 Webhook URL
   - 绑定个人微信：企业微信 → 我 → 设置 → 账号 → 微信插件 → 扫码

3. **配置 3 个 Secret**（仓库 Settings → Secrets → Actions）
   | 名称 | 值 | 说明 |
   |---|---|---|
   | `WECOM_WEBHOOK` | 你复制的 Webhook URL | 必填 |
   | `GEMINI_API_KEY` | https://aistudio.google.com/apikey 创建 | 强烈推荐 |
   | （可选）Telegram 通道 | 见下方说明 | 备选 |

4. **验证**：Actions 页面 → Market News Push → Run workflow → 看手机收没收到

## 两个运行模式

系统自动检测，你配了 Gemini Key 就用 AI 模式，没配就用关键词模式：

| | 有 Key（AI 模式） | 无 Key（关键词模式） |
|---|---|---|
| 评分方式 | Gemini 语义理解 1-10 分 | 关键词匹配，每个命中 +2 分 |
| 推送阈值 | ≥ 5 分 | ≥ 1 个关键词 |
| 方向判断 | 自动（利好/利空/中性） | 默认中性 |
| 效果 | 精准，理解上下文 | 粗糙，只靠字面匹配 |

> AI 模式强烈推荐。Google Gemini 免费层 1500 次/天，本项目每月只用约 500 次，完全在免费范围。

## 自选股配置

编辑 `config/watchlist.json`：

```json
[
  {
    "code": "sh600519",       // 股票代码，sh=沪市 sz=深市
    "name": "贵州茅台",         // 显示名称
    "market": "sh",
    "alerts": {
      "price_change_pct": 3,  // 涨跌幅超过 3% 就告警
      "volume_surge": 2,      // 量比超过 2 倍就告警
      "monitor_announcements": true  // 是否监控公告
    }
  }
]
```

**告警规则**：
- `price_change_pct: 3` → 涨跌超过 3% 触发
- `volume_surge: 2` → 成交量和量比同时超过 2 倍触发
- `monitor_announcements: true` → 新公告推送（年报、季报、减持、增持等）

**代码格式**：
- 沪市：`sh` + 6 位代码（`sh600519`）
- 深市：`sz` + 6 位代码（`sz000858`）
- 创业板：`sz` + 6 位代码（`sz300750`）

## （可选）配置 Telegram 备选通道

1. Telegram 搜 `@BotFather` → `/newbot` → 创建机器人 → 拿到 Token
2. 搜索你的机器人 → 发 `/start`
3. 浏览器打开 `https://api.telegram.org/bot<你的Token>/getUpdates`，复制 `chat.id`
4. GitHub 仓库 Secrets 添加：
   - `TELEGRAM_BOT_TOKEN`: 你的 Token
   - `TELEGRAM_CHAT_ID`: 你的 chat.id

## 修改关键词

编辑 `config/keywords.json` 的 `high_priority` 数组。关键词匹配模式下，每条命中的关键词给新闻 +2 分：

```json
{
  "high_priority": [
    "降息", "加息", "涨停", "跌停", "退市", "重组", "并购",
    "减持", "暴雷", "立案", "处罚", "黑天鹅"
  ],
  "score_threshold": 5
}
```

> AI 模式下 `score_threshold` 用来决定 AI 返回的多少分以上才推送。默认 5 分。

## 常见问题

### Q: 推送收到了但全是旧新闻/重复的？
A: 正常。首次运行时 data/sent.db 是空的，会把当前所有符合条件的新闻都推一边。跑过几轮后 SQLite 缓存建立，就不会重复了。90 天自动清理旧记录。

### Q: 财联社/东方财富数据抓不到了？
A: 免费 API 可能偶尔不稳定或接口变动。系统设计为双源并行，单源失败不影响另一路。如果两个都挂了，检查 GitHub Actions 日志看具体错误。

### Q: Gemini 评分返回空/异常？
A: 检查：
1. GEMINI_API_KEY 是否有效（在 AI Studio 验证）
2. 网络——GitHub Actions Runner 在境外，Google 服务无墙
3. 查看 Actions 日志里的 `[AI评分]` 开头的日志

### Q: 怎么确认它在跑？
A: GitHub Actions 页面两个 workflow 的 history 会记录每次运行。绿色打勾 = 成功，红色叉 = 失败，点进去看详细日志。

### Q: 会花我钱吗？
A: 全免费。GitHub Actions 公开仓库无限免费用；数据源全是公开 API；Gemini 免费层 1500 次/天足够；企微机器人免费。

### Q: 消息太多了刷屏怎么办？
A: 调高 AI 评分阈值（改 `src/main.py` 中 `>= 5` 改为 `>= 7`）或自选股告警阈值（改 `config/watchlist.json` 里的 `price_change_pct`）。
